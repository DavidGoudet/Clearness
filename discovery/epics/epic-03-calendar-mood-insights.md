# Epic 03 -- Calendar and Mood Insights

**Priority:** Must Have (MVP)
**Estimated Effort:** M (Medium)
**Target Duration:** 4-6 weeks

---

## Description

Build the Calendar tab that gives users a monthly overview of their mood history and enables them to review past journal entries. The calendar displays mood emojis on days where a chat was completed, highlights the current day, supports month-to-month navigation, and opens a daily summary view when a day is tapped. This feature transforms individual daily reflections into a visible pattern over time, which is central to the product's promise of "gaining clarity."

---

## Business Value

- Provides the feedback loop that makes daily journaling feel rewarding ("I can see my pattern")
- Drives re-engagement by giving users a reason to open the app even on days they have already journaled
- Mood pattern visibility supports the core value proposition of building self-awareness
- The daily summary view reinforces the value of past reflections, increasing perceived worth of continued use

---

## Acceptance Criteria (Epic Level)

1. The Calendar tab displays a standard monthly calendar grid for the current month
2. Days with a completed chat show the mood emoji that was selected during that day's chat
3. Days without a completed chat are blank (no emoji)
4. The current day is visually highlighted (distinct border or background)
5. Users can navigate to previous and next months using arrow controls
6. Tapping a day with a completed chat opens a daily summary showing: mood emoji and label, the user's free-text response, and the date and time of the chat
7. Tapping a day without a chat shows a "No entry for this day" placeholder
8. The calendar loads mood data efficiently (batch fetch per month, not per day)
9. The UI follows the Clearness design system and is usable on mobile viewports

## Child Story Candidates

| # | Story Title | Description | Size |
|---|-------------|-------------|------|
| 1 | Monthly mood summary API endpoint | Create GET /api/chats/monthly/?year=YYYY&month=MM endpoint that returns a list of {date, mood_emoji, mood_label} for all completed chats in the given month | 3 |
| 2 | Daily summary API endpoint | Create GET /api/chats/{date}/summary/ endpoint that returns mood, user response text, and completion timestamp for a specific date (or 404 if no chat exists) | 2 |
| 3 | Calendar grid component | Build the monthly calendar grid UI component that renders day cells in a 7-column layout with proper weekday headers and empty cells for offset days | 5 |
| 4 | Mood emoji overlay on calendar days | Integrate the monthly mood data into the calendar grid so that each day cell displays its corresponding mood emoji when data exists | 2 |
| 5 | Current day highlighting | Apply a visual highlight (teal-blue border or subtle background) to the cell representing today's date | 1 |
| 6 | Month navigation controls | Add previous/next month arrows and a month/year label; navigating fetches mood data for the target month | 3 |
| 7 | Daily summary view | Build the summary view (modal, bottom sheet, or dedicated panel) that displays when a calendar day is tapped: mood emoji and label, user's reflection text, date and time | 3 |
| 8 | Empty day placeholder | When a day without a chat is tapped, show a "No entry for this day" message in the summary view | 1 |
| 9 | Calendar data caching | Cache monthly mood data on the frontend so that navigating back to an already-loaded month does not trigger a redundant API call | 2 |
| 10 | Navigate from calendar to journal entry | Tapping a "View full chat" link in the daily summary navigates the user to the Journal tab showing that day's completed chat in read-only mode | 3 |

**Total estimated story points:** ~25

## Dependencies

| Epic | Dependency Type | Detail |
|------|----------------|--------|
| Epic 01 -- Authentication and Onboarding | Hard | User must be authenticated to access calendar data |
| Epic 02 -- Journal: Daily Reflection Chat | Hard | The calendar displays mood data generated by the journal chat flow; without completed chats, the calendar has no data to show |

---

## Risks and Considerations

- Performance: users who have journaled for months will generate large datasets; the monthly batch API must be efficient and paginated by month
- Timezone handling must be consistent with Epic 02; a chat's date determines which calendar cell it appears in
- The daily summary view must handle the case where chat data is encrypted (per PRD Section 5.1) -- decryption must happen before display
- Navigation between Calendar and Journal tabs (via "View full chat") requires shared routing state

---

## Definition of Done

- All acceptance criteria are met
- Calendar renders correctly for months with varying numbers of days (28, 29, 30, 31)
- Month navigation works across year boundaries (December to January)
- API endpoints have automated tests
- UI is responsive and matches the Clearness design system on mobile viewports
